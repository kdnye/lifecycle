How to Process a Lifecycle Event

1. Select the Event Type

Navigate to the main Intake Form. Begin by selecting whether this is an Onboarding (New Hire), Offboarding (Termination/Exit), or a Change/Transfer event. This selection determines the baseline logic the Rules Engine will apply.

2. Define Identity & Role

Provide the individual's legal names. Crucially, select the correct Role Profile (e.g., Driver, Office/Ops, Manager). The form will dynamically adapt based on this selection. For instance, selecting "Driver" will hide irrelevant questions about docking stations and internal VPNs.

3. Review the Dynamic Output

As you fill out the intake form, the system will actively preview the tasks it is preparing to generate. Verify that the expected hardware and access requirements match the role.

4. Submit & Auto-Provision

Upon submission, the system executes the following automatically:

  • Creates a master tracking record in the FSI Database.
  • Generates standard FSI database identity credentials.
  • Fires specialized, template-driven HTML emails via Postmark directly to our MSP ticketing systems.
Note on Offboarding: When offboarding, selecting "Immediate" priority will flag the resulting Stellar ticket as critical, prompting immediate Active Directory disablement and Microsoft 365 session revocation.
